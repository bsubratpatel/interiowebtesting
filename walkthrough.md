# Walkthrough of Next.js Performance Optimizations

I have completed the performance optimizations to minimize main-thread work, reduce JavaScript execution and script parsing times, and increase the Lighthouse Performance score to 95+.

## Summary of Changes

### 1. Dependencies and Bundle Cleanup
- Removed the unused `@radix-ui/react-navigation-menu` dependency from `package.json`.
- Ran `npm install` to prune node_modules, removing **18 unused packages** and significantly reducing total vendor chunk sizes.

### 2. React Server Component (RSC) Refactoring
- Removed static sections (`AboutSection`, `ServicesSection`, `CTASection`, `ProcessSection`, `TestimonialsSection`, `ContactSection`, `Footer`) from dynamic client-side bundles.
- Rendered them as **Server Components** directly in `src/app/page.tsx` (and `layout.tsx` for Footer) wrapped in `LazySection`. These sections are now compiled to static HTML on the server and ship **zero bytes of client-side JavaScript**.
- Created `LazyClientComponents.tsx` to group dynamic imports (`ssr: false`) for the two interactive components (`GallerySection` and `MaterialsSection`).
- Wrapped below-the-fold static section `WhyChooseUsSection` in `LazySection` to defer its DOM mounting until scrolled into view.

### 3. Scroll Interactions & DOM Cycle Optimization
- Refactored `FloatingCTAGroup.tsx` to prevent mount/unmount cycles during scroll:
  - Kept the component DOM nodes permanently mounted.
  - Toggled visibility using CSS transitions (`opacity`, `transform`) and `pointer-events-none`.
  - Throttled the scroll event state updates to only fire when the visibility state changes.
- Refactored `BackToTopButton.tsx` similarly to remain mounted, using CSS transition classes to toggle visibility smoothly without unmounting/mounting.

### 4. Stylesheet Injection Cleanup
- Extracted keyframe animation rules (`fadeIn`, `pulseCustom`, `fadeInReal`) and class definitions (`.animate-fade-in`, `.animate-pulse-custom`, `.animate-fade-in-150`) from `GallerySection.tsx` inline `<style>` block and appended them to the global stylesheet `src/app/globals.css`. This avoids runtime CSS parsing and layout recalculation when the gallery loads.
